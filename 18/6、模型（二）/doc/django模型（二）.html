• 模型元选项
    • 文档： http://python.usyiyi.cn/documents/django_182/ref/models/options.html
    • 每个model都可以定义一个Meta类，这个类中可以定义一些关于你的配置
    • Meta是一个model内部的类

class Meta:
        db_table = "topic"
        managed = True
        ordering = ['-id']
        verbose_name = u"主题"
        verbose_name_plural = u"主题列表"

  • db_table：string，  你在数据库中的表名
  • managed： bool， 是否通过migrate命令进行数据库关联管理
  • ordering:  数组,  默认排序规则
  • verbose_name: string model对象的描述
  • verbose_name_plural: string 复数时的描述

  • 外键字段
    • ForeignKey（多对一关系，关联模型关联的类）
class Retry(models.Model):
    topic = models.ForeignKey(Topic)
  • ForeignKey 外键的属性
    • db_constraint：bool  是否建立外键约束
    • to_field：string 关联到的关联对象的字段名称。默认地，Django 使用关联对象的主键。
    • related_name: string 这个名称用于让关联的对象反查到源对象.如果你不想让Django 创建一个反向关联，请设置related_name 为 '+' 或者以'+' 结尾.    
    • on_delete: string 可以取如下值
      • CASCADE:级联删除，如果删除，相关联的那个也会删除
      • PROTECT：保护类型。如果删除，将会抛出一个ProtectedError错误
      • SET_NULL：如果删除了本条数据，外键的那条数据将会设置为null，这个只有在外键null为True的情况下才可以使用
      • SET_DEFAULT：如果删除了本条数据，外键那条数据将会职位默认值，这个只有在外键那个字段设置了default参数才可以使用
  • ManyToManyField （多对多关系）

ps： self代表自己

假设我们现在有此对象
class Entry(models.Model):
    blog = models.ForeignKey(Blog)
    headline = models.CharField(u'大标题', max_length=255)
    pub_date = models.DateTimeField(u'出版时间')
    authors = models.ManyToManyField(Author)
  • all() 获取所有
Entry.objects.all()
  • filter(**kwargs)方法：根据参数提供的提取条件，获取一个过滤后的QuerySet。
Entry.objects.filter(headline=u'标题')
ps： 这里返回的不是最终的对象，是一个QuerySet

  • exclude(**kwargs)方法：根据参数提供的条件，排除符号条件的数据，返回一个QuerySet对象。
Entry.objects.exclude(headline=u'标题')

  • get(**kwargs)方法：只会返回一个对象，如果所有条件都不满足或者是满足条件的有多个，将抛出一个异常。
Entry.objects.get(pk=1)

  • order_by(*args)：根据给定的参数进行排序：

正序：
Entry.objects.filter(headline=u'标题').order_by('pub_date')
倒序：
Entry.objects.filter(headline=u'标题').order_by('-pub_date')


  • values()方法：将返回来的QuerySet中的Model转换为字典
Entry.objects.all().values()

  • count()方法：将返回当前查询到的数据的总数，这个比length方法更有效
Entry.objects.count()

  • latest(field_name=None)方法：根据提供的参数field_name来进行排序，field_name这个参数必须是时间字段，然后提取离现在最近的一条数据
Entry.objects.latest('pub_date')

  • earliest()方法：用法和latest一样，只是这个是获取最久远的一个。
Entry.objects.latest('pub_date')

  • first()方法：获取查询到的数据的第一条数据。如果用了order_by，那么将获取排序后的第一条。如果没有用order_by，那么将根据id进行默认排序。
Entry.objects.first()

  • last()方法：用法和first一样，只不过是获取的是最后一条的数据。
Entry.objects.first()

  • update(**kwargs)：更新数据方法。这个方法可以对查询出来的QuerySet里面的所有元素进行更新，并且更新参数的个数也是不限的。另外要注意的是，因为get方法返回的不是QuerySet对象，因此使用get方法提取出来的数据不能使用update方法。出于这种情况，建议应该使用filter(pk=xx)来替代get(pk=xxx)方法。并且，使用get出来的模型，修改数据后再save，会更新所有的数据，比update的效率更低。
Entry.objects.filter(id=1).update(headline="大标题")

  • delete()方法：删除QuerySet中的模型。
Entry.objects.filter(id=1).delete()

  • 查找对象的条件
                查找对象的条件的意思是传给以上方法的一些参数。比如name__contains=’abc’这个就代表name这个字段包含了abc的意思，相当于是SQL语句中的where语句后面的条件，语法为字段名__规则，以下将对这些规则进行说明：
  • exact：相当于等于号

Entry.objects.filter(headline__exact=u'大标题')

  • iexact：跟exact，只是忽略大小写的匹配。

Entry.objects.filter(headline__iexact=u'大标题')


  • contains：字符数据中包含等号后面的数据。
Entry.objects.filter(headline__contains=u'标题')

  • icontains：跟contains，唯一不同是忽略大小写

Entry.objects.filter(headline__icontains=u'标题')


  • in：判断字符的数据是否处在一个给定的列表中，如果在，则提取出来
Entry.objects.filter(id__in=[1, 2, 3])

  • gt：大于。
Entry.objects.filter(id__gt=1)

  • gte：大于等于

Entry.objects.filter(id__gte=1)

  • lt：小于。
Entry.objects.filter(id__lt=10)

  • lte：小于等于

Entry.objects.filter(id__lte=10)


  • startswith：以什么开始
Entry.objects.filter(headline__startswith='title')

  • istartswith：同startswith，忽略大小写
Entry.objects.filter(headline__istartswith='title')

  • endswith：同startswith，以什么结尾。
Entry.objects.filter(headline__iendswith='le')

  • iendswith：同istartswith，以什么结尾忽略大小写。

Entry.objects.filter(headline__iendswith='le')


  • range：区间查询
import datetime
start_date = datetime.date(2005, 1, 1)
end_date = datetime.date(2018, 1, 1)
Entry.objects.filter(pub_date__range=(start_date, end_date))


  • isnull：判断是否是空
Entry.objects.filter(pub_date__isnull=True)

  • 切片：对查找出来的数据进行切片，跟用数组是一样的
Entry.objects.all()[:2]

  • 外键、多对多模型之间的访问
    • 假设我们有以下模型关系
class Blog(models.Model):
    name = models.CharField(u'标题', max_length=100)
    content = models.TextField(u'内容')

class Author(models.Model):
    name = models.CharField(u'姓名', max_length=200)
    email = models.EmailField(u'邮箱')

class Entry(models.Model):
    blog = models.ForeignKey(Blog)
    headline = models.CharField(u'大标题', max_length=255)
    pub_date = models.DateTimeField(u'出版时间')
    authors = models.ManyToManyField(Author)
    
  • 访问外键：Entry的对象要访问他所持有的blog对象，通过以下代码即可访问：
entry.blog

  • 外键反向访问：如果Blog对象想要访问所有引用了他的Entry对象，可以通过“模型名_set“进行访问，例如：
blog.entry_set.all()
或者
blog.entry_set.filter(headline__contains=’I love’)

  • 多对多访问：跟外键访问一样，“模型名_set“进行访问。

entry.authors
